# 在线考试系统 V1.0 — 项目成熟度评估与改进清单

> - **评估对象**：smart-exam-system V1.0（前端 Vue3 + Element Plus；后端 Spring Boot 3 + JdbcTemplate）
> - **评估日期**：2026-06-07
> - **评估方式**：全栈代码静态审查（前端 `frontend/src` 全部组件与 API、后端 `controller/service/dto`、`schema.sql`、鉴权与路由机制）
> - **严重程度分级**：致命（核心流程不可用/数据丢失/安全漏洞）｜严重（重要功能缺失或明显错误）｜一般（功能不完善/体验问题）｜轻微（打磨细节）

---

## 评估综述

| 维度 | 成熟度 | 一句话结论 |
|---|---|---|
| 核心链路可用性 | ★★★☆☆ | 组卷→考试→阅卷→成绩主流程已打通，但答题中断不可恢复、防作弊缺失 |
| 功能完整度 | ★★☆☆☆ | 三端均停留在“基础 CRUD”，缺大量管理系统标配能力 |
| 权限与角色模型 | ★★☆☆☆ | 仅 3 个硬编码角色，管理员/教师职能严重重叠，数据隔离不一致 |
| 导航与信息架构 | ★★☆☆☆ | 无路由框架，刷新即回首页、前进后退失效 |
| 工程健壮性 | ★★☆☆☆ | 内存会话、无分页、AI 为假实现、防作弊形同虚设 |

**总体定位：可演示的 MVP，距离生产级“成熟系统”有系统性差距。**

## 一、三端功能矩阵对比

### 1.1 现有功能点

| 端 | 已具备功能 |
|---|---|
| **管理员** | 角色概览首页、班级管理、科目管理、知识点管理、公告管理、题库管理、试卷管理、用户管理（列表/启禁用/重置密码/删除）、角色与权限矩阵查看、系统日志、全局成绩分析 |
| **教师** | 概览首页、科目管理、知识点管理、公告管理、题库管理、试卷管理（手动/规则组卷）、考试任务（发布/编辑/结束/删除）、阅卷管理（主观题 + AI 辅助）、教师成绩分析 |
| **学生** | 概览首页、公告查看、考试中心（参加/倒计时/答题卡）、成绩查询、错题本、知识点掌握图 |

### 1.2 三端均缺失的通用基础功能

| 功能 | 现状 | 严重程度 | 改进建议 |
|---|---|---|---|
| 消息/通知中心 | 完全没有（公告是单向只读） | 严重 | 站内信表 + 通知中心 + 角标 |
| 数据导出（Excel/CSV） | 完全没有 | 严重 | 列表与成绩单接入导出 |
| 批量操作 | 完全没有（用户/题/卷均逐条） | 一般 | 表格多选 + 批量删除/启禁/导入 |
| 全局搜索 | 完全没有 | 一般 | 顶部全局搜索框，跨实体检索 |
| 个人中心/改密 | 用户无法修改自己密码（仅管理员能重置他人） | 严重 | 增加“个人中心”含改密、资料维护 |
| 打印 | 无打印成绩单/试卷 | 一般 | 成绩单/试卷打印视图 |

### 1.3 “应该有但完全不存在”的功能（重点）

| 缺失功能 | 严重程度 | 说明 |
|---|---|---|
| 教师查看**单个学生历次成绩趋势/分析** | 致命级缺口 | 后端 `StudentService` 全部按“当前登录学生”查，教师无任何按学生 id 查询的接口 |
| 导出**学生成绩单** | 严重 | 无任何导出 |
| 学生**标签/分组** | 严重 | 无标签体系，班级是唯一维度 |
| 对学生发送**个人评语/辅导建议** | 严重 | 阅卷有 comment，但无独立师生反馈通道 |
| **班级学生名单**管理 | 严重 | 教师看不到自己班有哪些学生 |
| 管理员**主动创建教师/学生账号** | 严重 | 仅支持自助注册，`UserController` 不能新建/改资料/调角色 |
| 考试**成绩排名/及格率报表** | 一般 | 仅有平均分与分数段，无排名 |
| 题目/试卷**导入** | 一般 | 只能手工录入 |

## 二、角色权限与职能边界混乱

### 2.1 管理员/教师功能重叠（逐端点核实）

| 端点/页面 | ADMIN | TEACHER | 问题 |
|---|---|---|---|
| 科目管理 `/api/basic/subjects` 增删改 | ✅ | ✅（`requireAnyRole`） | 职责重叠 |
| 知识点管理 `/api/basic/knowledge-points` | ✅ | ✅ | 职责重叠 |
| 公告管理 `/api/basic/notices` | ✅ | ✅ | 教师可发全局公告，易越权 |
| 题库 `/api/question-bank/**` | ✅ | ✅ | 职责重叠 |
| 试卷 `/api/papers/**` | ✅ | ✅ | 职责重叠 |
| 全局成绩分析 `/api/analysis/overview` | ✅ | ✅ | 教师可看全局统计 |

> **混乱点**：管理员与教师在“基础数据 + 题库 + 试卷”上几乎是同一套权限，管理员沦为“全能教师”，没有真正的“管理”与“教学”职责分层。班级管理（ADMIN only）是少数清晰的边界。

### 2.2 数据隔离缺失

| 位置 | 现象 | 严重程度 | 改进建议 |
|---|---|---|---|
| `QuestionBankService.listQuestions` | **无 `created_by` 过滤**，任意教师可见/改/删**所有教师**的题目 | 严重 | 按创建人或学科组隔离，或显式“共享题库”设计 |
| `PaperService.listPapers` | 同上，试卷全教师共享可改 | 严重 | 同上 |
| `ExamService.listTeacherExams` | **有** `created_by` 过滤 | — | 与题库/试卷**策略不一致**：同系统两套隔离逻辑 |
| `BasicDataService` 增删改 | 教师可删别的教师建的科目/知识点 | 严重 | 加归属或限制删除权 |

### 2.3 角色颗粒度

| 问题 | 严重程度 | 说明 |
|---|---|---|
| 仅 ADMIN/TEACHER/STUDENT 三个**硬编码**角色 | 严重 | `MenuService.allMenus/rolePageMap` 写死；`RoleController` 只读不可增 |
| 缺子管理员、年级主任、学科组长、教务等细分角色 | 严重 | 权限模型不可扩展 |
| 无“角色-权限点”动态配置 | 严重 | 权限硬编码，改权限需改代码重新部署 |

## 三、页面拓扑与导航故障

### 3.1 路由清单（无 vue-router，App.vue 手动映射）

`/login`、`/admin`、`/teacher`、`/student`（三端首页=概览）、`/basic/classes|subjects|knowledge-points|notices`、`/question-bank`、`/papers`、`/system/users`、`/system/roles`、`/monitor/logs`、`/exam/analysis`、`/exam-tasks`、`/reviews`、`/teacher/analysis`、`/student/exams|results|wrong-questions`。

### 3.2 故障清单

| 问题 | 复现/根因 | 严重程度 | 改进建议 |
|---|---|---|---|
| **刷新后强制回首页** | 任意后台页 F5 → `App.vue: onMounted → restoreSession → navigateTo(defaultPath)`，**恢复逻辑只用 `defaultPath`，完全不读当前 URL `location.pathname`**；影响**全部**后台页 | 严重 | 引入 vue-router，或 restoreSession 读取当前 path 恢复 |
| 与 Session/Token 关系 | **非 Token 丢失**：token 存 localStorage 仍在，是纯前端恢复逻辑丢了路径 | — | 同上 |
| 浏览器前进/后退失效 | 用 `history.replaceState`（非 pushState）且无 `popstate` 监听 | 严重 | pushState + popstate 或路由框架 |
| 面包屑不可点击 | topbar `crumb` 是纯文本 | 一般 | 面包屑做成可导航 |
| 无多页签 | 不支持 | 轻微 | 可选：多页签工作区 |
| **答题中刷新不可恢复** | `ExamTaking` 重新 `startExam`，后端 `status != 0` 抛错 → “进入考试失败”，已答内容全失 | 致命 | 区分“开始/继续”，按 `start_time` 计算服务端剩余时间 |
| 乱输 URL 行为 | `SpaController` 把所有非 API 路径 forward 到 index.html，SPA 内 `navigateTo` 不匹配菜单 → 静默回首页，无 404 提示 | 一般 | 增加 404 页 |

## 四、侧边栏结构与信息架构

### 4.1 菜单树（以管理员为例，叶子级）

```
管理员首页(/admin)
班级管理(/basic/classes)             ┐
科目管理(/basic/subjects)            │ 同级 4 项，实际都渲染同一个
知识点管理(/basic/knowledge-points)  │ BasicDataPanel 的不同 Tab
公告管理(/basic/notices)             ┘
题库管理(/question-bank)
用户管理(/system/users)
角色管理(/system/roles)
系统日志(/monitor/logs)
全局成绩分析(/exam/analysis)
试卷管理(/papers)
```

### 4.2 结构问题

| 问题 | 严重程度 | 改进建议 |
|---|---|---|
| 班级/科目/知识点/公告：菜单是 **4 个同级独立项，实际是 1 个 Tab 页面**（`BasicDataPanel`），菜单与实现不一致 | 一般 | 合并为“基础数据管理”父菜单，内部 Tab |
| 菜单**完全扁平**，无分组/父子层级（11 项平铺） | 一般 | 按“教学/系统/统计”分组折叠 |
| 排序无明确逻辑（成绩分析夹在中间、试卷管理排最后却与题库相关） | 一般 | 按业务流程“基础数据→题库→试卷→考试→阅卷→分析”重排 |
| 不支持侧边栏**收起/折叠** | 一般 | 加折叠按钮 |
| 不支持**深浅色模式**切换 | 轻微 | 加主题切换 |
| 不支持**拖拽排序/自定义常用入口** | 轻微 | 可选增强 |
| 菜单图标已在 `MenuService` 定义（icon 字段）但前端**未渲染** | 轻微 | 渲染 Element Plus 图标 |

## 五、数据查看粒度与教师工作流断层

### 5.1 教师下钻路径模拟

| 步骤 | 是否可行 |
|---|---|
| 查看所授班级 | ❌ 无“我的班级”页面 |
| 选中某班 → 查看学生列表 | ❌ 无班级学生名单接口/页面 |
| 点击某学生 → 个人档案 | ❌ 无学生档案页 |
| 查看该生考试记录与成绩变化图 | ❌ 无任何按学生 id 查询的教师接口 |
| 返回 | ❌ 路径根本不存在 |

> **结论：教师“以学生为中心”的工作流完全断裂，只能看到班级/全局聚合，无法触达任何一个具体学生。**

### 5.2 只有整体、无法下钻的页面

| 页面 | 现象 | 严重程度 |
|---|---|---|
| 教师成绩分析 | 仅各科平均分、分数段分布，无学生明细 | 严重 |
| 知识点掌握 | `StudentService.getKnowledgePointMastery` 只算“当前学生自己” | 严重 |
| 阅卷管理 | 按待批阅 attempt 列出，无“按学生归集” | 一般 |

### 5.3 缺失的数据维度

学生个人成绩对比、历年/历次变化曲线、错题本与考试关联分析、知识点薄弱项雷达图（仅学生自己有柱状图）、个人进步曲线、班级内排名 —— **均缺失**。

## 八、碎片问题池（逐条罗列）

| # | 位置 | 现象 | 严重程度 |
|---|---|---|---|
| 1 | `AiService.callAi` | **AI 是假实现**：配了 key 也只 `return "Real AI response for: " + prompt`，从未真正调用 OpenAI；但状态显示 "REMOTE" 误导为真 | 严重 |
| 2 | `ExamTaking` | 倒计时基于客户端 `durationMinutes*60`，非服务端 `start_time` 推算，理论上可被规避 | 严重 |
| 3 | 全站 | 防作弊形同虚设：`MonitorController` 有上报接口，但前端无 `monitor` API、`ExamTaking` 无切屏/复制监控 | 严重 |
| 4 | `StudentPanel.finishExam` | hack 写法：用切 Tab + `setTimeout` 强制刷新考试列表 | 一般 |
| 5 | 全站时间显示 | 直接渲染后端原始时间字符串（`createdAt/startTime/submitTime`），无统一格式化 | 一般 |
| 6 | `RoleOverviewController` | 概览卡片含“阶段 3 / 阶段 5 / 预留”等**开发阶段痕迹**，生产不应出现 | 一般 |
| 7 | 登录页 | 展示“后端/AI 状态”等调试性信息，偏演示风格 | 轻微 |
| 8 | `App.vue` | 顶层 `takingExam/startExam/finishExam` 与 `StudentPanel` 内部重复一套，实为死代码 | 轻微 |
| 9 | `ExamService.submitExam` | 客观题判分 `equalsIgnoreCase`，多选靠前端 `sort().join('')` 对齐，空格/格式敏感，判分脆弱 | 严重 |
| 10 | `QuestionBankService.listQuestions` | 列表对每行再查一次选项（N+1 查询） | 一般 |
| 11 | `exam.status` 字段 | 创建后恒为 0，无实际状态机，前端靠时间猜“进行中/已结束” | 一般 |
| 12 | `SystemLog` 的 action | AOP 泛化命名，学生答题被记成“新增考试”等不准确 | 轻微 |
| 13 | 知识点 `point_name` 唯一约束 | 软删后无法重建同名（uk 不含 deleted），与软删模型冲突 | 一般 |
| 14 | 空数据占位 | `el-empty` 各组件用法不完全一致，部分表格无空态 | 轻微 |
| 15 | 表单交互 | BasicDataPanel 用内联表单、UserManagement/ExamManagement 用 Dialog，新增/编辑交互不统一 | 轻微 |
| 16 | echarts 图表 | 无 `resize` 监听，窗口缩放图表不自适应 | 轻微 |
| 17 | Token 会话 | 内存 `ConcurrentHashMap`，服务重启或多实例部署 → 全部用户掉线 | 严重 |
| 18 | 注册开放性 | 任何人可自助注册教师账号（选 TEACHER 即可），无审核 | 严重 |

## 改进优先级建议

- **P0 致命/严重（先修）**：答题刷新不可恢复 + 答题自动保存；刷新回首页（引入路由）；列表分页；数据隔离（题库/试卷按归属）；AI 假实现澄清或补真；Token 持久化（JWT/Redis）；注册审核 + 登录失败限制。
- **P1 重要缺口**：教师“班级→学生→个人档案→成绩趋势”完整下钻链；管理员主动建号/改资料/派角色；成绩单导出；消息通知。
- **P2 体验与架构**：菜单分组/折叠/图标、面包屑、异常页、统一时间格式、防作弊上报、抽公共组件、角色权限动态化。
