# 273. 发布接口即时返回通知触达统计

## 背景

第 271 步后端已经记录发布触达统计，第 272 步前端审批日志也能查看这些统计。但管理员完成“直接发布”或“批准发布”后，成功提示仍只显示发布成功，操作人必须再打开审批记录才能知道本次发布实际触达了多少学生。

## 本步改动

- 管理员直接发布考试时，`createExam` 响应附带发布触达统计。
- 管理员批准教师考试申请时，`approveExam` 响应附带发布触达统计。
- 前端 `ExamInfo` 类型增加 `publishCandidateCount`、`publishNotifiedStudentCount`、`publishNotifiedAttemptCount`。
- 考试管理页和考试审批队列页的发布成功提示展示候选考生数、通知学生数和生成答卷数。
- 质量门增加后端响应字段与前端成功反馈检查。

## 三端协同价值

- 管理员端在发布动作完成的第一时间获得触达确认，不需要再进入日志二次核对。
- 教师端后续仍可通过审批记录追溯管理员批准后的触达统计。
- 学生端入口、通知和答卷模板生成结果能被操作人即时确认，减少“已发布但学生没收到”的排查成本。

## 验收点

- 管理员直接发布考试后，接口响应包含三项发布触达统计。
- 管理员批准发布后，接口响应包含三项发布触达统计。
- 成功提示展示“候选 / 通知 / 答卷”三类结果。
- 审批日志仍保留可追溯的持久化统计。
