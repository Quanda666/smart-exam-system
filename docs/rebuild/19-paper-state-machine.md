# 19. 试卷状态机保护

## 目标

本批次把试卷从“可随时编辑的题目集合”收紧为考试系统中的稳定资产。试卷一旦发布或被考试引用，不能再直接修改题目结构、分值、科目、名称或删除，避免已发布考试和历史答卷被后续编辑污染。

## 后端变更

- `PaperService` 为创建、更新、状态切换、删除、规则组卷增加事务边界。
- 创建/更新试卷时先按草稿写入题目和题目版本快照，只有发布校验通过后才切换到发布态。
- 发布试卷必须满足：
  - 试卷存在且未删除；
  - 至少包含一道题；
  - 题目均未删除、已启用、审核状态为 `APPROVED`；
  - `paper_question.question_version_id` 指向有效的 `question_version` 快照。
- 已发布试卷禁止结构性更新和删除。
- 被任意未删除考试引用的试卷禁止结构性更新、删除和撤回。
- 试卷列表和详情返回操作元数据：
  - `examCount`
  - `locked`
  - `lockReason`
  - `canEdit`
  - `canDelete`
  - `canPublish`
  - `canRevoke`

## 前端变更

- 试卷列表新增锁定状态列，展示 `Locked/Editable` 和引用考试数量。
- 编辑、发布/撤回、删除按钮根据后端返回的 `can*` 字段禁用。
- 点击已锁定试卷的编辑/删除/状态切换时给出明确提示。
- 批量发布、撤回、删除仅处理当前允许操作的试卷，避免单条锁定数据导致整批失败。

## 三端协作影响

- 管理员端和教师端看到同一套试卷可操作状态，避免不同入口规则不一致。
- 教师发布考试时仍只能选择已发布试卷；试卷被考试引用后自动成为稳定快照来源。
- 学生端答题使用考试快照，不受后续题库变更影响；本批次进一步阻止试卷本体在考试引用后被改动。

## 后续任务

- 增加“复制为新试卷”接口，让教师可以基于锁定试卷创建新草稿。
- 引入 `PaperStatus` 枚举替代当前 `0/1` 状态值。
- 后续若支持试卷多版本，应新增 `paper_version` 和 `paper_version_question`，让考试引用试卷版本而不是试卷主表。
