# 09 部署与运维说明

## 环境分层

- 本地环境：Docker Compose 启动 MySQL、后端、前端。
- 测试环境：独立数据库、Redis、对象存储，用于联调和自动化测试。
- 生产环境：至少两个应用实例、独立 MySQL、Redis、对象存储、日志系统。

## 高可用改造

- Token 会话迁移到 Redis，支持多实例。
- 作答草稿优先写 Redis 和 MySQL，Redis 用作高频缓存，MySQL 做最终持久化。
- 上传资料、申诉附件、导出文件放对象存储，不依赖本地磁盘。
- 考试交卷使用事务写入答卷和答题记录，避免半提交。

## 监控指标

- 应用：请求量、错误率、P95、JVM 内存、线程池、GC。
- 数据库：连接池、慢 SQL、锁等待、磁盘容量。
- Redis：连接数、内存、命中率、慢命令。
- 考试：进入考试人数、作答中人数、草稿保存失败数、交卷失败数。
- 监考：异常事件量、事件上报失败量。

## 备份恢复

- MySQL 每日全量备份，每小时增量或 binlog 备份。
- 对象存储开启版本控制。
- 关键配置导出为版本化文件。
- 每月至少演练一次恢复到测试环境。

## 发布策略

- 后端先跑数据库迁移，再滚动发布应用。
- 前端静态资源使用版本 hash，避免缓存旧代码。
- 发布前必须通过自动化测试和冒烟测试。
- 考试进行中禁止执行破坏性数据库变更。
