# 170. 监考事件风险分可见化

## 背景

第 169 批已经把单条监考事件的 `risk_score` 持久化到后端，并在事件 API 中返回 `riskScore`。但教师端和管理员端的事件明细仍主要展示事件类型与附加信息，风险分没有形成可见闭环。

本批把事件级风险分贯通到监考事件明细和导出文件，方便教师查看某一次行为的风险贡献，也方便管理员做审计留档。

## 本批改动

- 教师端监考事件时间线新增风险分标签：
  - 使用后端返回的 `event.riskScore`。
  - 使用现有 `riskType` 颜色规则，低/中/高风险视觉口径与会话总风险一致。
- 新增 `.event-title` 布局，保证事件名称和风险标签在小屏可换行展示。
- 监考事件 CSV 导出新增 `Risk Score` 表头。
- 导出每条事件时写入 `riskScore`。
- 质量门新增检查：
  - 后端导出必须包含 `Risk Score` 和 `event.get("riskScore")`。
  - 前端监考事件明细必须渲染风险分标签。

## 三端协同影响

- 学生端：无交互变化，仍不需要提交风险分。
- 教师端：打开某个学生的监考事件后，可以看到每条事件的风险分。
- 管理员端：导出的监考事件明细包含单条风险分，审计时不必重新按事件类型推算。

## 验收点

- 事件明细时间线显示 `Risk <score>` 标签。
- 导出的监考事件 CSV 包含 `Risk Score` 列。
- 导出数据行包含每条事件的 `riskScore`。
- 前端类型、后端接口和导出字段口径一致。
