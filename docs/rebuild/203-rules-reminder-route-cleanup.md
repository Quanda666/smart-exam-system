# 203. 规则提醒路由状态清理

## 目标

规则提醒通知已经可以直达学生考试中心。本批次继续补齐尾部状态清理，避免学生进入考试或考试结束后，页面仍因为旧的 `notice=rules` 查询参数持续显示提醒。

## 功能范围

- 学生点击 `Confirm rules` 并进入目标考试前，清理当前路由中的 `notice` 与 `attemptId`。
- 仅清理当前目标考试对应的规则提醒参数，不影响其他查询参数。
- 规则提醒条只在目标答卷未提交且 `rulesConfirmedAt` 为空时展示。
- 已提交或已完成规则确认的考试不再显示规则提醒条。

## 协同价值

- 教师端：发送规则提醒后，学生完成处理不会继续看到过期提醒。
- 学生端：通知落地、确认、进入考试、返回考试中心的体验更干净。
- 审计端：通知记录和处置记录保留，不依赖前端 query 表示是否已处理。

## 验收要点

- 点击规则提醒通知后仍能定位目标考试。
- 点击 `Confirm rules` 进入考试后，URL 中不再保留 `notice=rules&attemptId=<id>`。
- 已提交考试不展示规则提醒条。
- 已有 `rulesConfirmedAt` 的考试不展示规则提醒条。
