# 08 测试与验收说明

## 后端测试

- 使用 Testcontainers MySQL，避免依赖本机数据库。
- 覆盖认证、角色权限、数据范围、题库、试卷、考试发布、作答、交卷、阅卷、成绩发布。
- 状态机测试必须覆盖非法流转：未开始开考、已结束开考、重复交卷、越权阅卷、超分阅卷。
- 安全测试必须覆盖伪造题号、篡改 attemptId、学生访问题库、教师访问非授课学生。

## 前端测试

- 使用 Vitest 做工具函数和组件逻辑测试。
- 使用 Playwright 做三端端到端流程。
- 作答页测试刷新恢复、离线草稿、自动保存失败重试、时间到自动交卷。
- 路由守卫测试 401、403、404、考试中后退拦截。

## 端到端主流程

1. 管理员创建班级、课程、课程班、教师、学生。
2. 管理员分配教师授课并同步学生选课。
3. 教师创建题目、发布题目、组卷。
4. 教师发布考试到课程班。
5. 学生进入考试、作答、保存草稿、交卷。
6. 教师查看监考事件并批阅主观题。
7. 教师发布成绩。
8. 学生查看成绩、错题和解析。
9. 学生提交成绩申诉，教师处理。

## 压测验收

- 300 并发学生同时进入考试，错误率小于 1%。
- 草稿保存 P95 小于 500ms。
- 交卷 P95 小于 1500ms。
- 数据库连接池无耗尽，慢 SQL 可定位。
- Redis、MySQL、应用日志容量有告警。

## 人工验收

- 三端菜单无死链。
- 所有快捷入口都能进入正确页面。
- 所有高风险操作都有确认和日志。
- 考试状态、答卷状态、成绩可见性在三端显示一致。
