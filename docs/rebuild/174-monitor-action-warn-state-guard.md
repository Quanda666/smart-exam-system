# 174. 监考提醒动作状态约束

## 背景

监考处置动作里，`WARN` 会向学生发送提醒通知。此前后端只对 `FORCE_SUBMIT` 做了状态校验，`WARN` 可以对已交卷、已阅卷或已完成的会话发送。这会造成两个问题：

- 学生已交卷后收到“遵守考试纪律”的提醒，语义不合理。
- 教师端处置记录和学生端通知不再对应真实作答阶段。

本批把 `WARN` 纳入监考处置状态机：只允许对作答中的答卷发送提醒。

## 本批改动

- `validateMonitorActionState` 新增 `WARN` 校验：
  - 仅当 `exam_attempt.status == 1` 时允许。
  - 其他状态抛出 `WARN action requires an in-progress attempt`。
- `FORCE_SUBMIT` 原有约束保持不变：
  - 必须已经完成强制交卷。
  - `submit_type` 必须为 `FORCED`。
- `ACKNOWLEDGE` 和 `NOTE` 保持可用于历史会话审计。
- 教师端处置弹窗中，非作答中会话禁用 `WARN` 选项。
- 质量门新增后端状态机和前端禁用项检查。

## 三端协同影响

- 学生端：只会在仍处于作答中时收到监考提醒。
- 教师端：已交卷会话仍可关注或备注，但不能再发送实时提醒。
- 管理员端：处置审计里 `WARN` 语义更稳定，表示真实作答期间的提醒。

## 验收点

- 作答中会话可以记录 `WARN` 并发送提醒通知。
- 已交卷会话不能记录 `WARN`。
- 已交卷会话仍可记录 `ACKNOWLEDGE` 或 `NOTE`。
- 教师端非作答中会话的 `WARN` 选项不可选。
