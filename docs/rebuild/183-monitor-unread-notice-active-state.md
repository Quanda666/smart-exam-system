# 183. 未读监考通知快捷筛选状态反馈

## 背景

第 182 批把教师端监考面板的 `Unread notices` 指标改成快捷筛选入口。教师点击后虽然会切换到 `Latest notification = Unread`，但指标卡本身没有激活态，也没有在未读数为 0 时阻止无意义点击。

## 本批改动

- `Unread notices` 指标按钮增加 active 状态，和当前 `latestNotificationStatus = 'UNREAD'` 保持同步。
- 当当前筛选范围内未读通知数为 0 时禁用按钮，避免教师点出空结果后误以为系统异常。
- `applyUnreadNoticeFilter` 增加空数据保护，防止通过事件或后续改动绕过禁用态。
- 保留 `aria-pressed`，让快捷筛选按钮对辅助技术表达当前开关状态。
- 质量门禁增加 active、disabled、空数据保护等断言，避免后续回退。

## 三端协同影响

- 教师端：监考看板的未读通知筛选更明确，已启用和不可用状态一眼可见。
- 学生端：不改变通知送达、已读和考试作答流程。
- 管理员端：不改变通知审计查询接口；教师侧仍可用通知 ID 回查审计记录。

## 验收点

- 未读数量大于 0 时，点击 `Unread notices` 后下拉筛选切换为 `Unread`。
- `Latest notification = Unread` 时，指标按钮显示 active 状态。
- 未读数量为 0 时，指标按钮禁用且点击不会改变筛选条件。
- 既有状态、风险等筛选条件不会被快捷筛选清空。
