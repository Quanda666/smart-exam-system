# 160. 阅卷评分审计日志考试归属修复

## 背景

前两批修复了成绩申诉和申诉审计日志的考试归属。继续审计管理员审计链路后，`review_score_log` 也需要同样的保护：如果历史评分日志的 `exam_id` 与 `exam_attempt.exam_id` 不一致，管理员阅卷审计、导出和后续告警统计会挂到错误考试。

本批统一阅卷评分日志的归属规则：

- 能通过 `attempt_id` 找到答卷时，以 `exam_attempt.exam_id` 为准。
- 找不到答卷时，保留原有日志快照。

## 本批改动

- 启动迁移：修正 `review_score_log.exam_id` 为空或与答卷考试不一致的历史日志。
- 运行时写入：`recordReviewScoreLog` 写日志前通过 `attempt_id` 解析真实考试 ID。
- 管理员审计：现有阅卷评分日志列表和 CSV 导出继续读取 `review_score_log`，但数据来源会被迁移和写入路径校正。
- 质量门禁：固定迁移必须修正阅卷评分日志考试归属，固定写入必须解析答卷考试。

## 三端协同影响

- 教师端：每次人工阅卷评分生成的审计日志绑定真实考试。
- 管理员端：阅卷评分审计、筛选和导出不会因为历史错快照挂到错误考试。
- 学生端：无交互变化，但成绩复核和申诉时引用的阅卷轨迹更可信。

## 验收点

- 历史 `review_score_log.exam_id IS NULL` 的日志会按答卷考试回填。
- 历史 `review_score_log.exam_id <> exam_attempt.exam_id` 的日志会被修正。
- 新写入阅卷评分日志时，能找到答卷就使用 `exam_attempt.exam_id`。
- 管理员阅卷评分审计列表和导出继续基于修正后的日志考试 ID。
