# 186. 通知审计结果 ID 可见与复制

## 背景

教师端可以复制通知审计 ID，管理员端也可以按 ID 搜索通知审计。但搜索结果表格之前没有直接展示通知 ID，管理员确认记录后不方便把准确编号回传给教师或写入处置留档。

## 本批改动

- 管理员端 `Notification Audit` 结果表新增 `ID` 列。
- 每条通知审计记录显示 `#id`，和教师端 `Notification #id` 的表达保持一致。
- 每行新增复制按钮，复制内容为纯通知 ID。
- 复制逻辑优先使用 `navigator.clipboard.writeText`，并保留 textarea 后备方案。
- 增加局部样式，保证 ID 与复制按钮在表格中稳定对齐。
- 质量门禁增加 ID 列、复制按钮、剪贴板函数和样式断言。

## 三端协同影响

- 教师端：发送给管理员的通知 ID 可以被管理员在结果中直接确认。
- 管理员端：审计结果可以反向复制通知 ID，用于留档、沟通或二次查询。
- 学生端：不改变通知读取、考试作答或监考事件上报流程。

## 验收点

- `Notification Audit` 表格首列显示 `#id`。
- 点击复制按钮后，剪贴板内容为纯通知 ID。
- 搜索、导出和通知 ID 规范化行为不受影响。
- 通知审计导出仍包含通知 ID 字段。
