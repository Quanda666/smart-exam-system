# 96. 管理员端操作日志检索与导出

## 背景

系统日志是管理员追踪高风险操作、定位误操作和做上线验收的重要入口。此前操作日志只能分页查看，无法按操作者、动作、对象、时间范围检索，也不能导出归档。对于正式在线考试系统，这会让管理员在排查“谁改了试卷、谁发布了考试、谁强制交卷”时只能人工翻页。

## 本次改动

- 管理员操作日志列表接口新增筛选参数：
  - `keyword`：匹配操作人、动作、对象、详情或 IP。
  - `action`：按动作模糊匹配。
  - `target`：按对象模糊匹配。
  - `startFrom` / `startTo`：按操作时间范围过滤。
- 管理员端新增导出接口：
  - `GET /api/monitor/logs/export`
- 后端将操作日志筛选逻辑抽为 `appendOperationLogFilters`，列表和导出共用同一套口径。
- 导出字段包括：
  - 时间
  - 操作人
  - 动作
  - 对象
  - 详情
  - IP
  - 操作人 ID
  - 日志 ID
- 管理员“系统日志 / 操作日志”页新增筛选栏和“导出”按钮。
- 单次导出最多返回 5000 条记录，避免大范围误导出拖慢管理端。

## 协同关系

- 操作日志承接认证、用户管理、题库、试卷、考试、阅卷、监考等通用写操作审计。
- 管理员可以先用操作日志定位具体业务动作，再跳转到成绩发布审计、成绩申诉审计或监考记录做专项复盘。
- 该导出不替代专项审计表，只提供跨模块的全局操作追踪入口。

## 验收点

- 管理员可以按关键词、动作、对象和时间范围筛选操作日志。
- 操作日志导出遵守当前筛选条件。
- CSV 文件中文在 Excel 中可正常显示。
- 普通教师和学生不能访问操作日志列表和导出接口。
- 本地质量门禁覆盖操作日志筛选和导出入口。
