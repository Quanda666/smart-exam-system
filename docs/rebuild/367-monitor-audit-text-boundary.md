# 367. Monitor Audit Text Boundary

## 背景

监考事件和监考处置记录属于风险证据。学生端上报的 `extraInfo`、教师端处置 `note`、监考强制交卷备注都会进入日志、通知和后续审计链路。此前 DTO 已限制 1000 字符，但服务层仍会静默截断，内部调用或未来接口绕过校验时可能留下被改写的证据文本。

## 本步改造

- `MonitorService` 新增 `MAX_MONITOR_TEXT_LENGTH = 1000`。
- 新增 `normalizeMonitorText`，统一将空白文本归一化为 `null`，并拒绝超过 1000 字符的监考证据文本。
- 监考事件 `extraInfo`、监考处置 `note`、监考强制交卷备注全部改用该规范化方法。
- 保留强制交卷默认备注：未填写备注时仍使用 `Forced submission from monitor session`。
- 质量门新增监考文本边界检查，防止重新引入 `trimToLength(..., 1000)` 静默截断。

## 三端影响

- 学生端：超长监考事件附加信息会被明确拒绝，不会写入截断后的风险记录。
- 教师端：监考处置和强制交卷备注保持原文可信，超长备注需要用户调整。
- 管理员端：全局监控审计看到的文本与实际提交文本一致，不再混入被服务端截断的证据。

## 验收点

- 超过 1000 字符的 `extraInfo` 不能进入 `cheat_event`。
- 超过 1000 字符的监考处置备注不能进入 `exam_monitor_action`。
- 强制交卷空备注仍会落默认说明。
