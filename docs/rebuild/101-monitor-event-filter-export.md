# 101. 监考事件筛选与筛选导出

## 背景

第 99 批已经支持按答卷导出监考原始事件，第 100 批补齐了人工处置记录导出。但在真实考试复盘中，单个答卷可能产生大量风险事件，教师或管理员通常需要先按事件类型、时间窗口定位关键片段，再导出对应证据。

如果页面筛选和导出范围不一致，复盘材料会很容易产生歧义。因此本批把事件列表查询和事件导出统一接入同一套筛选参数。

## 本次改动

- `GET /api/monitor/cheat-events/{attemptId}` 新增可选筛选参数：
  - `eventType`
  - `startFrom`
  - `startTo`
- `GET /api/monitor/cheat-events/{attemptId}/export` 使用同一组筛选参数。
- 后端新增 `appendCheatEventFilters`：
  - `eventType` 仍受监考事件白名单约束。
  - `startFrom/startTo` 按服务端接收时间 `event_time` 过滤。
  - 时间格式兼容 ISO-8601、UTC instant、本地日期时间和日期。
- 前端监考事件抽屉新增筛选栏：
  - 事件类型下拉。
  - 时间范围选择。
  - 筛选与重置操作。
- 前端 `exportAttemptMonitorEvents` 接收 `MonitorEventQuery`，导出当前筛选结果。
- 本地质量门覆盖后端过滤、前端 query builder、抽屉筛选状态和筛选导出联动。

## 三端协同

- 学生端继续按批量上报原始监考事件，不承担筛选逻辑。
- 教师端在监考抽屉里先筛选事件，再查看详情或导出当前筛选结果。
- 管理员端复查同一答卷时，沿用相同接口权限与筛选规则。
- 事件筛选只改变复盘视图和导出范围，不改变风险分、答卷状态或违规结论。

## 验收点

- 监考事件抽屉可以按事件类型筛选。
- 监考事件抽屉可以按服务端接收时间范围筛选。
- 点击导出事件时，CSV 只包含当前筛选条件命中的事件。
- 非白名单事件类型会被后端拒绝。
- 没有答卷监考权限的教师仍不能查询或导出事件。
- 监考事件仍然只是风险记录，不自动判定违规。
