# 95. 管理员端成绩发布审计导出

## 背景

成绩发布和撤回是在线考试系统中最敏感的管理动作之一。第 84 批已经让管理员能查看全局成绩发布审计，第 94 批补齐了成绩申诉审计导出。本批继续补齐成绩发布审计的导出能力，让发布、撤回、影响答卷和通知送达数据可以归档。

## 本次改动

- 管理员端新增导出接口：
  - `GET /api/monitor/score-release-logs/export`
- 导出接口复用列表页筛选条件：
  - `keyword`
  - `action`
  - `startFrom`
  - `startTo`
- 后端将成绩发布审计列表的筛选逻辑抽为 `appendScoreReleaseAuditFilters`，列表和导出共用同一套口径。
- 导出字段包括：
  - 时间
  - 动作
  - 考试
  - 试卷
  - 状态流转
  - 处理人
  - 影响答卷
  - 通知学生
  - 通知答卷
  - 说明/原因
  - 考试 ID
  - 日志 ID
- 管理员“系统日志 / 成绩发布审计”页新增“导出”按钮。
- 单次导出最多返回 5000 条记录，避免大范围误导出影响系统稳定性。

## 协同关系

- 教师或管理员发布/撤回成绩后写入 `score_release_log`。
- 管理员在系统日志页查看和导出的成绩发布审计使用同一批数据、同一套筛选条件。
- 学生端成绩可见性、成绩撤回通知、申诉入口都可以引用这份导出作为发布/撤回动作的审计证据。

## 验收点

- 管理员可以导出成绩发布审计 CSV。
- 导出结果遵守页面筛选条件。
- 导出记录包含状态流转、影响答卷数和通知送达统计。
- CSV 文件中文在 Excel 中可正常显示。
- 普通教师和学生不能访问导出接口。
