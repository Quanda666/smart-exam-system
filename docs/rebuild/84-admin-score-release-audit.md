# 84. 管理员成绩发布全局审计

## 背景

第 83 步已经为成绩发布和撤回写入 `score_release_log`，但教师端只能按单场考试查看。管理员端需要跨考试审计成绩发布动作，支持排查“谁发布/撤回了成绩”“影响了多少答卷”“是否通知到学生”等问题。

## 本次改动

- 新增管理员接口 `GET /api/monitor/score-release-logs`。
- 支持分页、关键词、动作、时间范围筛选。
- 关键词覆盖考试名称、试卷名称、处理人姓名和账号。
- 查询结果包含考试、试卷、动作、状态流转、处理人、影响答卷数、通知学生数、通知答卷数、说明/原因和时间。
- 管理员端 `系统日志` 页面新增 `成绩发布审计` tab。
- 前端 API 新增 `ScoreReleaseAuditLog`、`ScoreReleaseAuditQuery`、`listScoreReleaseAuditLogs`。
- 质量门禁新增全局审计接口、服务查询、前端 API 和页面入口检查。

## 三端协同意义

- 教师端：单场考试内可查看成绩记录，便于本考试操作复盘。
- 管理员端：跨考试统一审计发布/撤回动作，适合安全巡检、异常排查和责任追踪。
- 学生端：后续成绩不可见、撤回通知和成绩申诉可以引用同一批审计流水。

## 验收点

- 管理员访问 `/monitor/logs` 能看到 `成绩发布审计` tab。
- 可按考试/试卷/处理人关键词筛选。
- 可按 `PUBLISH` / `REVOKE` 动作筛选。
- 可按时间范围筛选。
- 表格展示影响答卷数和通知数。
- 非管理员不能访问全局审计接口。
- 本地质量门禁通过。
