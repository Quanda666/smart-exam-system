# 100. 监考处置记录导出

## 背景

第 98 批已经支持按考试导出监考会话汇总，第 99 批已经支持按答卷导出原始监考事件。正式考试复盘还需要补齐另一类证据：教师或管理员对某个监考会话做过哪些人工处置，例如关注、提醒学生、备注观察结论、强制交卷等。

这些处置记录不能只停留在页面里，必须能够按单个学生会话导出，方便学院归档、申诉复核和考试纪律复盘。

## 本次改动

- 新增监考处置明细导出接口：
  - `GET /api/monitor/sessions/{sessionId}/actions/export`
- 导出接口复用现有监考会话处置权限：
  - 管理员可导出全局监考处置记录。
  - 考试创建者可导出本考试监考处置记录。
  - 非创建者教师只能导出自己授课范围内学生的处置记录。
- `loadSessionForAction` 补充导出上下文：
  - 考试名称
  - 学生姓名
  - 学号
  - 班级
- 导出字段包括：
  - 考试
  - 学生
  - 学号
  - 班级
  - 处置类型
  - 处置人
  - 处置时间
  - 处置说明
  - 处置 ID
  - 会话 ID
  - 答卷 ID
  - 考试 ID
- 前端实时监考抽屉新增“导出处置”按钮，和“导出事件”并列显示。
- 本地质量门新增后端接口与前端按钮/API 覆盖检查。

## 三端协同

- 学生端继续负责上报风险事件和接收监考提醒、强制交卷等通知。
- 教师端在实时监考面板中查看事件、记录处置，并可导出单个学生的处置记录。
- 管理员端通过全局监控和审计权限，可以复查监考事件、会话汇总和人工处置记录。
- 原始事件与人工处置分开导出：事件说明“发生了什么”，处置说明“监考人员如何响应”，两者共同构成监考复盘材料。

## 验收点

- 打开某个学生的监考事件与处置抽屉后，可以导出处置记录 CSV。
- CSV 至少包含处置类型、处置人、处置时间、处置说明、会话 ID 和答卷 ID。
- 没有授课范围权限的教师不能导出不属于自己学生的处置记录。
- 处置记录仍然只作为风险响应记录，不自动判定学生违规。
- 本地质量门覆盖后端导出接口、权限入口、前端 API 和按钮状态。
