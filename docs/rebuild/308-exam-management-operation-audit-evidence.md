# 308 考试任务管理操作审计证据

## 背景

考试任务是试卷、考生范围、答卷、阅卷和成绩发布之间的核心对象。成绩发布和审批动作已有独立日志，但教师或管理员在考试管理页创建、编辑、删除、提前关闭考试后，页面缺少本次操作对应的 `operation_log` 证据，无法像用户、基础数据、题库、试卷那样立即复制审计 ID。

## 本步范围

- `ExamController` 注入 `OperationLogService`。
- 考试任务写操作响应新增 `operationLogId`：
  - 创建考试：`CREATE_EXAM`
  - 编辑考试：`UPDATE_EXAM`
  - 删除考试：`DELETE_EXAM`
  - 提前关闭考试：`CLOSE_EXAM`
- `api/exam.ts` 在 `ExamInfo`、删除响应、关闭响应中声明 `operationLogId`。
- `ExamManagement.vue` 在单条和批量考试操作成功后展示最近一次操作审计证据。
- 支持复制审计 ID 和 `/monitor/logs?operationLogId=<id>` 深链。
- `scripts/run-quality-gates.ps1` 增加后端与前端防回归检查。

## 三端协同关系

- 教师端：创建或调整考试安排、提前关闭考试后，可立即复制审计证据交给管理员或教务核验。
- 管理员端：统一系统日志可以按 `operationLogId` 精确定位考试任务变更，和审批日志、成绩发布日志拼成完整生命周期证据链。
- 学生端：不直接看到操作审计信息，但考试入口、作答权限和成绩可见性都依赖考试任务状态；管理员可用审计证据追踪状态变更来源。

## 验收点

- 创建、编辑、删除、提前关闭考试接口均返回 `operationLogId`。
- 考试管理页对应操作成功后出现审计提示条。
- 批量删除能收集多个 `operationLogId` 并展示。
- 复制审计 ID 和复制统一日志深链均可用。
- 本地质量门禁覆盖后端返回与前端展示复制能力。
