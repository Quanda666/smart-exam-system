# Step 355 - Review Attempt ID Boundary

## 背景

教师阅卷详情、提交阅卷、查看评分日志、导出评分日志都以 `attemptId` 作为入口。此前非法 `attemptId` 会进入阅卷权限查询，再表现为“答卷不存在”或“无权限”等业务错误，参数边界不够清晰。

阅卷写入点已经具备待阅卷状态、教师数据范围、完整覆盖、分数上下限和审计日志约束；本次补齐的是公开入口的答卷 ID 参数边界。

## 本次改动

- `ReviewService.requireReviewAccess` 统一调用 `requirePositiveAttemptId`。
- 新增 `requirePositiveAttemptId`，拒绝 `attemptId == null` 或 `attemptId <= 0`。
- 质量门新增结构性断言，要求阅卷服务在访问权限查询前拒绝非正数答卷 ID。

## 协同意义

- 教师端：非法阅卷链接会得到稳定参数错误，不会混入权限或业务状态错误。
- 后端：阅卷详情、提交、日志列表和日志导出共享同一入口边界。
- 管理员端：阅卷审计链路只围绕有效答卷展开，减少异常排查噪声。

## 验收点

- `attemptId <= 0` 不能进入阅卷权限查询。
- 阅卷详情、提交、评分日志列表、评分日志导出都共享该校验。
- 已有待阅卷状态、分值上限、完整覆盖和评分日志行为不变。
