# 272. 审批日志发布触达统计前端展示

## 背景

第 271 步已在后端审批日志中落库并返回考试发布触达统计，包括候选学生数、成功通知学生数和绑定答卷数。前端审批记录仍只显示动作、状态、处理人、时间和备注，管理员或教师无法从页面确认一次发布是否真正触达学生。

## 本步改动

- 扩展 `ExamApprovalLog` 前端类型，接收 `candidateCount`、`notifiedStudentCount`、`notifiedAttemptCount`。
- 在考试管理页审批记录抽屉增加“发布触达”列。
- 在考试审批队列记录抽屉增加同样的“发布触达”列。
- 仅对 `APPROVE` 和 `DIRECT_PUBLISH` 展示统计，其它提交、重提、驳回动作显示 `-`。
- 更新质量门，防止后续改动遗漏审批日志触达统计展示。

## 三端协同价值

- 管理员端可以在审批队列和考试管理入口核对发布通知是否覆盖目标考生。
- 教师端可以通过考试管理入口确认管理员批准后实际生成的通知和答卷范围。
- 学生端虽不直接展示该统计，但其考试入口、通知和答卷模板生成结果现在能被管理端追踪，形成发布审计闭环。

## 验收点

- 后端返回审批日志时，发布动作行包含三项统计字段。
- 前端审批记录中，发布动作展示“候选 / 通知 / 答卷”三项数值。
- 非发布动作不展示无意义的 0 值。
- `scripts/run-quality-gates.ps1` 能检查类型和两个页面的展示逻辑。
