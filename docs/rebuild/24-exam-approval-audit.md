# 24. 考试审批意见与审计记录

## 目标

本批次在第 23 批“教师提交审批、管理员批准/驳回”的基础上，补齐审批意见和审批日志。考试发布不再只是状态变化，而是形成可追踪、可解释、可复盘的工作流。

## 后端变更

- 新增 `exam_approval_log` 表：
  - `exam_id`：考试 ID。
  - `action`：`SUBMIT`、`RESUBMIT`、`APPROVE`、`REJECT`、`DIRECT_PUBLISH`。
  - `status_from`、`status_to`：记录状态流转。
  - `note`：批准备注或驳回原因。
  - `actor_id`、`created_at`：记录处理人和处理时间。
- 新增 `ExamApprovalDecisionRequest`：
  - `note` 最大 1000 字。
  - 批准时可选，驳回时服务端强制必填。
- `POST /api/exams/{id}/approve`：
  - 支持批准备注。
  - 审批通过后写入 `APPROVE` 日志。
  - 给考试创建人发送审批通过通知。
- `POST /api/exams/{id}/reject`：
  - 必须填写驳回原因。
  - 状态变为 `REJECTED` 后写入 `REJECT` 日志。
  - 给考试创建人发送驳回通知。
- `PUT /api/exams/{id}`：
  - 已驳回考试被教师修改保存后自动回到 `PENDING_APPROVAL`。
  - 写入 `RESUBMIT` 日志。
- 新增 `GET /api/exams/{id}/approval-logs`：
  - 管理员可查看所有考试审批记录。
  - 教师只能查看自己创建或授课范围可见考试的审批记录。

## 前端变更

- 考试管理页新增状态筛选：
  - 待审批
  - 已发布
  - 已关闭
  - 已驳回
- 管理员批准考试时可填写批准备注。
- 管理员驳回考试时必须填写驳回原因。
- 考试操作列新增“审批记录”入口。
- 审批记录抽屉展示：
  - 动作
  - 状态流转
  - 处理人
  - 处理时间
  - 备注/原因

## 三端协同影响

- 管理员端具备审批留痕能力，不再只能看到当前状态。
- 教师端收到驳回通知后，可以查看驳回原因，修改后重新提交。
- 学生端仍只消费 `PUBLISHED` 考试；审批日志不会暴露给学生。

## 后续任务

- 增加管理员审批队列独立视图，默认只展示 `PENDING_APPROVAL`。
- 在教师端考试详情页增加审批时间线，减少教师反复进入管理表格查看状态。
- 增加审批 SLA 指标，例如待审批时长、平均审批耗时、驳回率。
- 将审批日志纳入系统操作日志，支持按管理员、教师、考试周期审计导出。
