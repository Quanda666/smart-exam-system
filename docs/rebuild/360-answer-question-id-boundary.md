# Step 360 - Answer Question ID Boundary

## 背景

学生作答链路是考试闭环的核心入口。系统已经按考试快照或试卷题目集合校验提交答案和草稿答案的题目归属，也会在交卷时补齐未答题记录。但如果提交答案或草稿 JSON 中包含 `0`、负数或空题号，此前会被归类为“题目不属于本次答卷”，参数错误和业务范围错误没有明确区分。

正式在线考试系统需要在进入题目范围匹配前先拒绝非法题号，确保保存草稿、提交答卷、未答题补齐、自动判分和后续阅卷都基于可信题目 ID。

## 本次改动

- `AnswerRequest.answers` 的 Map key 增加元素级 `@Positive`，提交答案中的题号必须为正数。
- `ExamService.validateSubmissionAnswers` 在匹配答卷题目集合前调用 `requirePositiveAnswerQuestionId`。
- `ExamService.validateDraftAnswersJson` 在解析草稿 JSON key 后、匹配题目集合前调用同一正数边界。
- 新增 `requirePositiveAnswerQuestionId`，明确拒绝空题号和非正数题号。
- 质量门禁新增结构断言，防止提交答案和草稿保存退回到只依赖范围查询兜底。

## 协同意义

- 学生端：非法题号会得到稳定的参数错误，不会混入草稿恢复或交卷状态错误。
- 教师端：阅卷任务只基于合法题目记录生成，减少异常答题明细进入人工批阅。
- 管理员端：监控、审计、成绩统计和异常排查可以区分非法参数与真实越权/题目范围错误。

## 验收点

- 提交答案时，`answers` 中任一题号 `<= 0` 会被拒绝。
- 保存草稿时，草稿 JSON 中任一题号 `<= 0` 会被拒绝。
- 非本次答卷题目仍由原有题目范围校验拒绝。
- 历史草稿恢复的清洗策略不变，避免旧脏数据阻塞学生进入考试页。
- 合法提交、合法草稿、未答题补齐、自动判分和人工阅卷流程不变。
