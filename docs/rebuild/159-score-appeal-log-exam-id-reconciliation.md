# 159. 成绩申诉审计日志考试归属修复

## 背景

第 158 批修复了 `score_appeal.exam_id` 与答卷考试不一致的问题。继续审计后，发现管理员端申诉审计和导出读取的是 `score_appeal_log.exam_id`。如果历史日志的考试快照写错，审计记录会挂到错误考试上，影响管理员排查、导出归档和后续告警统计。

本批把申诉审计日志也纳入同一归属规则：

- 能通过 `attempt_id` 找到答卷时，以 `exam_attempt.exam_id` 为准。
- 找不到答卷时，才保留日志原有 `exam_id` 快照。

## 本批改动

- 启动迁移：修正 `score_appeal_log.exam_id` 为空或与答卷考试不一致的历史日志。
- 运行时写入：`recordAppealLog` 写日志前先通过 `attempt_id` 解析真实考试 ID。
- 管理员审计：现有监控审计和导出继续读取 `score_appeal_log`，但数据来源被迁移和写入路径校正。
- 质量门禁：固定迁移必须修正日志考试归属，固定日志写入必须解析答卷考试。

## 三端协同影响

- 学生端：提交申诉和查看申诉结果无交互变化。
- 教师端：处理、复核、关闭申诉产生的审计日志会绑定真实考试。
- 管理员端：全局申诉审计、筛选和 CSV 导出不会因为历史错快照把记录挂到错误考试。

## 验收点

- 历史 `score_appeal_log.exam_id IS NULL` 的日志会按答卷考试回填。
- 历史 `score_appeal_log.exam_id <> exam_attempt.exam_id` 的日志会被修正。
- 新写入申诉日志时，能找到答卷就使用 `exam_attempt.exam_id`。
- 管理员申诉审计列表和导出继续基于修正后的日志考试 ID。
