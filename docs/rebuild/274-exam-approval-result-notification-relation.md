# 274. 审批结果通知绑定考试关联

## 背景

学生的新考试通知已经绑定到 `EXAM_ATTEMPT`，可以通过通知审计追溯到具体答卷。但教师收到的“考试已批准/已驳回”通知仍是普通 `EXAM_APPROVAL` 通知，没有 `related_type` 和 `related_id`，管理员无法按考试维度核对教师是否收到了审批结果。

## 本步改动

- 审批通过和审批驳回通知继续使用通知类型 `EXAM_APPROVAL`。
- 通知新增业务关联：`related_type = EXAM`，`related_id = examId`。
- 保留非去重发送策略，避免同一考试多次驳回、重提、再驳回时丢失历史通知。
- 质量门增加断言，防止审批结果通知退回为无关联普通通知。

## 三端协同价值

- 管理员端可以在通知审计里按 `EXAM / examId` 查询教师审批结果通知。
- 教师端收到的审批结果通知可以和具体考试建立稳定关联。
- 学生端发布通知仍按 `EXAM_ATTEMPT` 绑定答卷入口，教师审批通知按 `EXAM` 绑定考试流程，两类通知边界更清晰。

## 验收点

- 批准发布后，教师通知记录包含 `related_type = EXAM` 和对应 `examId`。
- 驳回发布后，教师通知记录包含 `related_type = EXAM` 和对应 `examId`。
- 多次审批动作仍可产生多条历史通知，不因关联字段被错误去重。
- 管理员通知审计可按关联类型和关联 ID 筛选该考试通知。
