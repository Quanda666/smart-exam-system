# 第 74 批：成绩发布审计信息可见化

## 背景

第 73 批已经要求撤回成绩必须填写原因，并将原因写入 `score_release.note`。但如果这些信息只停留在接口响应和数据库里，教师/管理员在考试管理列表中仍然只能看到“已发布/未发布”这种粗粒度状态。

本批把成绩发布审计信息展示到考试管理页，让撤回原因和撤回时间能被直接查看。

## 本批改动

- 教师/管理员考试列表查询新增返回：
  - `scoreRevokedAt`：成绩撤回时间。
  - `scoreReleaseNote`：成绩发布/撤回备注，目前用于撤回原因。
- 前端 `ExamInfo` 类型补充：
  - `scoreRevokedAt`
  - `scoreReleaseNote`
- 考试管理页成绩发布列从二态扩展为三态：
  - `已发布`
  - `已撤回`
  - `未发布`
- 成绩发布列增加 tooltip：
  - 已发布时展示发布时间。
  - 已撤回时展示撤回时间和撤回原因。
- 总质量门新增源码护栏，防止后端列表字段或前端审计展示被回退。

## 三端协同影响

- 管理员端：能直接看到某场考试成绩是否曾被撤回，以及撤回原因。
- 教师端：撤回原因不再只在操作后短暂提示或数据库中存在，后续进入考试管理页仍可追溯。
- 学生端：继续通过 `SCORE_REVOKED` 通知接收撤回原因，本批不改变学生端页面。

## 验收方式

本地质量门：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run-quality-gates.ps1
```

人工验收建议：

1. 发布一场考试成绩。
2. 撤回成绩并填写原因。
3. 回到考试管理列表。
4. 成绩发布列应显示“已撤回”。
5. 鼠标悬停该状态，应看到撤回时间和撤回原因。

## 后续建议

- 增加专门的成绩发布审计抽屉，展示发布人、发布时间、撤回人、撤回时间、撤回原因、通知触达人数。
- 将 `score_release.note` 拆分为 `publish_note` 和 `revoke_reason`，避免未来发布备注和撤回原因语义混用。
- 在成绩撤回后禁用“导出成绩单”或在导出文件中标记成绩已撤回。
