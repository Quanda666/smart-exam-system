# 23. 考试发布审批流

## 目标

本批次把“教师创建考试即发布给学生”的高风险流程拆成“教师提交审批 -> 管理员批准 -> 学生可见”的正式发布链路。管理员仍可直接发布考试，用于教务统一安排或补录历史考试。

## 状态定义

- `0 = PENDING_APPROVAL`：待审批。教师提交后进入该状态，学生不可见，不生成考生答卷。
- `1 = PUBLISHED`：已发布。管理员批准后进入该状态，生成考生快照、试卷快照、初始答卷和通知。
- `2 = CLOSED`：已关闭。考试提前结束后进入该状态，进行中的答卷会被强制交卷。
- `3 = REJECTED`：已驳回。管理员驳回后进入该状态，学生不可见，教师可编辑后重新进入待审批。

## 后端变更

- `POST /api/exams`：
  - 教师创建考试时只保存考试安排和参考范围，状态为 `PENDING_APPROVAL`。
  - 管理员创建考试时在预检通过后直接生成快照并发布。
- `POST /api/exams/{id}/approve`：
  - 仅管理员可调用。
  - 仅允许审批 `PENDING_APPROVAL` 考试。
  - 审批时重新读取考试安排和目标范围，执行发布预检。
  - 通过后生成试卷题目快照、考生范围快照、初始答卷，并发送考试通知。
- `POST /api/exams/{id}/reject`：
  - 仅管理员可调用。
  - 仅允许驳回 `PENDING_APPROVAL` 考试。
  - 驳回后学生端仍不可见，也不会生成作答入口。
- `PUT /api/exams/{id}`：
  - 待审批考试可编辑。
  - 已驳回考试可编辑，保存后自动回到 `PENDING_APPROVAL`。
  - 已发布考试仍只允许在未开始、无人进入时编辑。
- 学生端考试同步只读取 `PUBLISHED` 考试，待审批和已驳回考试不会进入学生考试中心。

## 前端变更

- `ExamManagement` 接收当前用户主角色：
  - 管理员按钮文案为“发布考试”。
  - 教师按钮文案为“提交审批”。
- 管理员考试列表对 `PENDING_APPROVAL` 记录展示“批准”和“驳回”操作。
- 教师考试列表展示“待审批”“已驳回”等状态。
- 已驳回考试允许重新编辑；保存后由后端恢复为待审批。
- 已发布、已开始或已有学生进入的考试继续禁止编辑和删除。

## 三端协同

- 管理员端负责考试发布闸口：检查时间、试卷、参考范围、考生数量和题目快照后再批准。
- 教师端负责编排考试内容和目标范围：提交后等待管理员处理，驳回后修正并重新提交。
- 学生端只消费正式发布结果：只有管理员批准后的考试才会生成作答入口、监考会话和通知。

## 后续任务

- 增加审批意见字段，记录驳回原因和批准备注。
- 增加审批日志表，保留每次提交、驳回、重新提交、批准的操作者和时间。
- 增加管理员审批队列筛选页，支持按教师、课程班、考试时间、风险项过滤。
- 增加教师端审批状态时间线，让教师明确看到谁处理、何时处理、为何驳回。
- 将考试状态数字迁移为正式枚举字段说明，并补充数据库约束。
