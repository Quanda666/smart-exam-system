# 366. Audit Text Boundary

## 背景

审批备注和成绩撤回原因会进入审批日志、成绩发布日志、通知内容和后续申诉证据链。接口层已经有长度限制，但服务层仍保留成绩撤回原因的静默截断逻辑，内部调用或未来接口改动可能写入被改写后的审计文本。

## 本步改造

- `ExamService` 新增 `MAX_SCORE_REVOKE_REASON_LENGTH = 500` 和 `MAX_APPROVAL_NOTE_LENGTH = 1000`。
- `scoreRevokeReason` 不再 `trimToLength(reason, 500)`，超长撤回原因直接拒绝。
- 新增 `normalizeApprovalNote`，审批通过备注和驳回理由统一修剪空白、空串归一化为 `null`，并拒绝超过 1000 字符的文本。
- 质量门新增 DTO 与服务层双重文本边界检查，防止审计文本重新退回静默截断。

## 三端影响

- 管理员端：审批/驳回和成绩撤回的超长文本会收到明确错误，不会产生被截断的日志。
- 教师端：成绩撤回原因保持完整可信，便于后续复核和申诉沟通。
- 学生端：成绩撤回通知中的原因与审计日志一致，不再存在“通知看到的是截断文本”的风险。

## 验收点

- 超过 500 字符的成绩撤回原因不能进入 `score_release` 或 `score_release_log`。
- 超过 1000 字符的审批备注/驳回理由不能进入 `exam_approval_log`。
- 空白审批备注按 `null` 处理，驳回理由仍必须非空。
