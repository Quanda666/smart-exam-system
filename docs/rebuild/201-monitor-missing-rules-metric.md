# 201. 规则确认缺失监控指标

## 目标

教师端已经支持按规则确认状态筛选，并能对缺失确认的学生发送规则提醒。本批次把“规则未确认”提升为监考台首屏指标，让教师进入考试监控后能立即发现并处理准入确认异常。

## 功能范围

- 监考指标区新增 `Missing rules` 指标卡。
- 指标统计当前筛选结果中 `rulesConfirmedAt` 为空的会话数量。
- 指标卡可点击，点击后自动应用 `rulesConfirmationStatus = MISSING`。
- 当没有缺失规则确认的会话时，指标卡禁用。
- 激活 Missing rules 筛选时，指标卡保持高亮状态。

## 协同价值

- 学生端：规则确认仍以服务端 `exam_attempt.rules_confirmed_at` 为准。
- 教师端：从“发现缺失确认”到“筛选学生”再到“发送 Rules reminder”形成连续操作。
- 管理员端：后续可复用同一指标做全局考试准入异常看板。

## 验收要点

- 有未确认规则会话时，`Missing rules` 数字大于 0 且可点击。
- 点击后，规则筛选下拉值切换为 `Missing`。
- 列表只保留规则确认时间为空的会话。
- 重置筛选后，指标卡取消激活，规则筛选回到 `All rules`。
