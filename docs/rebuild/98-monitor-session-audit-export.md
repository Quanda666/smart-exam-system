# 98. 监考会话审计导出

## 背景

实时监考看板已经能展示学生在线状态、风险事件、风险分和教师处置记录。但正式考试结束后，教务和管理员还需要按考试归档监考情况，用于复盘断网、切屏、提醒学生、强制交卷等过程。此前这些信息只能在页面查看，缺少稳定导出能力。

## 本次改动

- 新增监考会话导出接口：
  - `GET /api/monitor/exams/{examId}/sessions/export`
- 导出接口复用现有监考数据范围：
  - 管理员可导出全局考试监考会话。
  - 教师只能导出自己创建考试或授课范围内学生的监考会话。
- 后端导出复用 `listExamMonitorSessions` 的结果口径，保证看板和 CSV 数据一致。
- 导出字段包括：
  - 考试
  - 学生
  - 学号
  - 班级
  - 在线/离线/已交卷状态
  - 风险等级
  - 风险分
  - 事件数
  - 最后事件
  - 最后心跳
  - 最近处置
  - 处置人
  - 处置时间
  - 交卷状态
  - 交卷方式
  - 交卷时间
  - 答卷 ID
  - 会话 ID
- 前端实时监考页新增“导出”按钮，按当前选中考试下载监考会话 CSV。

## 协同关系

- 学生端上报监考事件，后端聚合到 `exam_monitor_session`。
- 教师端在实时监考页查看风险并记录处置。
- 管理员或教师导出的 CSV 作为考试结束后的监考复盘材料。
- 作弊事件仍只作为风险记录，不自动判定违规；最终处理仍需人工依据制度确认。

## 验收点

- 选择考试后可以导出监考会话 CSV。
- 导出内容与实时监考表格的风险、事件和处置口径一致。
- 无权限教师不能绕过接口导出不属于自己范围的监考会话。
- CSV 文件中文在 Excel 中可正常显示。
- 本地质量门禁覆盖后端导出接口和前端导出按钮。
