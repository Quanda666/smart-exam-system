# 22. 考试生命周期状态机

## 目标

本批次把考试从“只靠时间判断阶段”推进为后端强约束的生命周期。当前阶段先保留现有数字状态，明确 `1=PUBLISHED`、`2=CLOSED`，并收紧编辑、删除、关闭、成绩发布和学生进入规则。

## 后端变更

- `ExamService` 增加状态常量：
  - `EXAM_STATUS_PUBLISHED = 1`
  - `EXAM_STATUS_CLOSED = 2`
- 创建考试仍在预检通过后直接进入发布态，但统一使用状态常量。
- 学生进入、保存草稿、提交答卷继续要求考试处于 `PUBLISHED`。
- 提前结束考试时：
  - 将 `exam.status` 更新为 `CLOSED`；
  - 将 `end_time` 收敛到当前时间；
  - 对进行中的答卷使用已保存草稿强制交卷，未保存内容按未答处理。
- 编辑考试安排时必须满足：
  - 考试仍处于发布态；
  - 当前时间未到开始时间；
  - 没有学生进入过考试。
- 删除考试时必须满足：
  - 当前时间未到开始时间；
  - 没有学生进入过考试。
- 发布成绩前必须满足：
  - 考试已关闭或结束时间已过；
  - 没有进行中的答卷；
  - 仍沿用既有的“无待阅卷、存在已完成答卷”校验。
- 教师考试列表新增：
  - `startedAttemptCount`
  - `activeAttemptCount`

## 前端变更

- 考试列表按 `status=2` 显示“已关闭”。
- 编辑、删除、提前结束按钮根据生命周期状态禁用。
- 批量删除自动过滤不可删除考试。
- 操作函数入口增加二次提示，减少旧列表数据误操作。

## 三端协作影响

- 教师端不能再修改已开始或已有学生进入的考试安排。
- 学生端在考试关闭后无法继续进入、保存或提交普通答卷。
- 教师提前结束考试会收口进行中的答卷，后续成绩发布不再卡在“进行中”状态。

- 增加管理员发布审批入口，把教师创建考试从“直接发布”拆为“提交审批 -> 发布”。
- 增加关闭考试审计日志，记录关闭人、关闭原因和被强制交卷人数。
