# 94. 管理员端成绩申诉审计导出

## 背景

第 93 批让管理员能在系统日志中查看跨考试、跨学生、跨处理人的成绩申诉审计流水。但真实项目里，审计记录还需要用于线下归档、复盘会议、责任追踪和外部检查，仅页面查看不够。

## 本次改动

- 管理员端新增导出接口：
  - `GET /api/monitor/score-appeal-logs/export`
- 导出接口复用列表页筛选条件：
  - `keyword`
  - `action`
  - `handlingResult`
  - `startFrom`
  - `startTo`
- 后端使用现有 `CsvExport` 和 `ExportFile` 生成带 UTF-8 BOM 的 CSV，避免 Excel 打开中文乱码。
- 导出字段包括：
  - 时间
  - 动作
  - 考试
  - 学生
  - 学号
  - 申诉对象
  - 状态流转
  - 处理结果
  - 操作人
  - 说明
  - 申诉 ID
  - 答卷 ID
- 管理员“系统日志 / 成绩申诉审计”页新增“导出”按钮。
- 前端导出使用当前筛选条件，不受当前分页限制。
- 单次导出最多返回 5000 条记录，避免误操作拖垮管理端。

## 协同关系

- 学生端提交申诉、教师端处理/复核、管理员端审计导出使用同一张 `score_appeal_log`。
- 管理员导出的记录和系统日志页面查询口径一致，避免页面审计与归档报表不一致。
- 该导出不修改成绩、不触发通知，仅作为审计证据留存。

## 验收点

- 管理员可以导出成绩申诉审计 CSV。
- 导出结果遵守页面筛选条件。
- 导出记录包含状态流转和结构化处理结果。
- CSV 文件中文在 Excel 中可正常显示。
- 普通教师和学生不能访问导出接口。
