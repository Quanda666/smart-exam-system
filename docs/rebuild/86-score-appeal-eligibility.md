# 86. 成绩申诉准入与窗口提示

## 背景

第 85 步让学生端能区分成绩已发布、待发布和已撤回，但申诉入口还缺少更清晰的窗口状态。后端已经要求成绩已发布且答卷完成后才能提交申诉，本次把这条规则显式暴露给学生端，避免学生到提交时才收到拒绝。

## 本次改动

- `GradeInfo` 新增：
  - `appealOpen`
  - `appealDeadlineAt`
  - `appealWindowDays`
- 学生成绩列表和详情按系统配置 `score.appealEnabled`、`score.appealWindowDays` 计算申诉窗口。
- 学生端结果详情展示申诉截止时间。
- 申诉窗口关闭、成绩未发布、成绩撤回时禁用整卷/单题申诉入口。
- `openAppeal` 增加二次保护，防止绕过按钮状态。
- 保留后端 `ScoreAppealService.submitAppeal` 的硬约束：只允许已发布、已完成、未超期成绩提交申诉。

## 状态语义

- `appealOpen = true`：学生可提交整卷或单题申诉。
- `appealDeadlineAt != null`：展示具体截止时间。
- `appealWindowDays <= 0`：申诉长期开放，前端显示长期开放。
- 成绩未发布、已撤回、待批阅、待判分：不可申诉。

## 验收点

- 已发布且未超期成绩可以提交申诉。
- 已发布但超期成绩禁用申诉入口。
- 成绩未发布或已撤回时不能提交申诉。
- 前端显示申诉截止时间或长期开放提示。
- 后端仍拒绝伪造未发布成绩申诉请求。
- 本地质量门禁通过。
