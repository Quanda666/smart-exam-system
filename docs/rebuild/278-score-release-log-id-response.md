# 278. 成绩发布与撤回响应返回审计日志 ID

## 背景

成绩发布和撤回已经会写入 `score_release_log`，并记录影响答卷数、通知学生数和通知答卷数。但操作完成后的接口响应和前端提示只展示数量，管理员或教师需要再打开成绩记录抽屉才能找到本次操作对应的审计记录。

## 本步改动

- `recordScoreReleaseLog` 返回插入的 `score_release_log.id`。
- `publishScores` 响应增加 `scoreReleaseLogId`。
- `revokeScores` 响应增加 `scoreReleaseLogId`。
- 前端成绩发布和撤回成功提示展示“成绩记录 #ID”。
- 质量门检查后端响应字段、前端类型和成功提示消费。

## 三端协同价值

- 管理员端和教师端完成成绩发布/撤回后，可以立即记录本次审计编号。
- 学生端仍按成绩发布状态控制可见性，通知仍关联 `EXAM_ATTEMPT`，不破坏答卷级通知追踪。
- 后续若要从成绩记录跳转通知审计或导出单次发布报告，已有稳定的日志 ID 作为入口。

## 验收点

- 发布成绩后响应包含本次 `scoreReleaseLogId`。
- 撤回成绩后响应包含本次 `scoreReleaseLogId`。
- 前端成功提示显示该审计编号。
- 既有成绩发布记录列表仍展示历史日志。
