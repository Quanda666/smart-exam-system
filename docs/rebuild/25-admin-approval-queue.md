# 25. 管理员考试审批队列

## 目标

本批次把管理员审批从“考试管理列表里的几个按钮”升级为独立审批工作台。管理员默认聚焦待审批考试，可以按教师、考试时间、状态和风险项快速筛选，并在同一页面完成批准、驳回和审批记录查看。

## 后端变更

- 新增 `GET /api/exams/approvals`：
  - 仅管理员可访问。
  - 支持 `keyword`、`creatorKeyword`、`status`、`startFrom`、`startTo`、`risk`、`page`、`size`。
  - 返回考试基础信息、创建教师、参考范围数量、题目数量、总分、待审批时长、最近审批意见和风险标记。
- 审批队列风险项：
  - `PAST_START`：待审批考试的开考时间已过。
  - `NO_TARGET`：没有参考范围。
  - `NO_QUESTIONS`：试卷没有题目。
  - `PASS_SCORE_OVER_TOTAL`：及格线超过试卷总分。
- 队列排序：
  - 待审批优先。
  - 开考时间更早的优先。
  - 同时间下新申请靠前。

## 前端变更

- 新增管理员页面 `/exam-approvals`。
- 管理员菜单新增“考试审批”，归入“考试与题库”分组。
- 角色授权页面新增 `/exam-approvals` 可选权限。
- 新增 `ExamApprovalQueue`：
  - 顶部指标：待处理、风险项、当前页申请、筛选结果。
  - 筛选条件：考试/试卷/科目、教师、状态、风险项、开考时间范围。
  - 表格字段：考试、教师、开考时间、范围/题目、分数、状态、风险、最近意见。
  - 待审批记录可直接批准或驳回。
  - 支持查看审批记录抽屉。

## 三端协同影响

- 管理员端具备集中审批入口，适合教务或学院管理员批量处理教师提交的考试。
- 教师端仍通过考试任务提交审批和查看驳回后的状态。
- 学生端仍只看 `PUBLISHED` 考试，审批队列不会暴露给学生。

## 后续任务

- 审批队列增加“只看高风险”快捷筛选和批量驳回草稿。
- 审批记录增加导出能力，纳入系统审计报表。
- 管理员仪表盘加入待审批数量和超时审批提醒。
- 增加审批 SLA 统计：平均审批耗时、最长待审时长、驳回率、教师重提次数。
