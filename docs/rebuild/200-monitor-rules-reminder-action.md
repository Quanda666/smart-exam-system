# 200. 规则确认缺失会话提醒闭环

## 目标

上一批已经能筛选和导出未完成规则确认的监考会话。本批次补齐教师端处置动作，让教师可以对规则确认缺失的学生发送专项提醒，并把提醒动作纳入监考处置审计。

## 功能范围

- 监考会话操作列新增 `Rules` 提醒入口。
- 新增监考处置动作 `RULES_REMINDER`。
- 后端仅允许对 `rules_confirmed_at` 为空且尚未结束的答卷发送规则确认提醒。
- 提醒会写入 `exam_monitor_action`，并发送学生通知。
- 通知类型为 `MONITOR_RULES_REMINDER`，关联对象为 `EXAM_ATTEMPT`。

## 协同链路

1. 学生端首次进入考试前应确认规则，后端写入 `exam_attempt.rules_confirmed_at`。
2. 教师端监考列表通过 `Rules = Missing` 找到异常会话。
3. 教师点击 `Rules`，系统生成 `RULES_REMINDER` 处置记录。
4. 学生收到考试规则确认提醒，回到考试中心继续完成规则确认。
5. 后续管理员可从监考处置记录和通知审计中追踪提醒是否发出、是否已读。

## 状态约束

- 已确认规则的会话不能发送 `RULES_REMINDER`。
- 已提交或已结束的答卷不能发送 `RULES_REMINDER`。
- `RULES_REMINDER` 是风险提醒和流程修正，不代表作弊判定。

## 验收要点

- 缺少规则确认且答卷未结束时，教师端 `Rules` 按钮可点击。
- 已确认规则或已提交的会话，`Rules` 按钮不可点击。
- 保存后，最近处置显示 `Rules reminder`。
- 处置详情中可看到提醒记录和通知审计 ID。
- 后端拒绝对已确认规则的会话直接调用 `RULES_REMINDER` 接口。
