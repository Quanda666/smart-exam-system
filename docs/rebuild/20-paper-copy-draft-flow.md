# 20. 试卷复制为新草稿

## 目标

第 19 批已经禁止已发布或已被考试引用的试卷直接编辑。本批次补齐教师端的后续工作流：当试卷被锁定后，教师可以复制为一份新的草稿，在新草稿上继续调整题目、分值和说明，再走发布流程。

## 后端变更

- 新增接口：`POST /api/papers/{id}/copy`。
- 复制权限保持保守：只有管理员或原试卷创建者可以复制。
- 复制出来的新试卷固定为草稿状态：
  - `status = 0`
  - `created_by = 当前用户`
  - `examCount = 0`
  - `locked = false`
- 新试卷名称自动生成：
  - 默认使用 `原试卷名 Copy`
  - 若重名，则追加序号
  - 名称长度控制在请求 DTO 限制内
- 复制题目时重新走当前组卷校验：
  - 题目必须存在；
  - 题目必须启用；
  - 题目审核状态必须为 `APPROVED`；
  - 复制后的 `paper_question.question_version_id` 绑定当前题目的最新版本。

## 前端变更

- `PaperInfo` 增加 `canCopy`。
- `paper.ts` 增加 `copyPaper(id)`。
- 试卷列表操作列增加“复制”按钮。
- 复制成功后自动把新草稿载入手动组卷工作台，教师可以立即继续编辑。

## 与状态机的关系

- 复制不是修改原试卷，因此不会破坏已发布考试和历史答卷。
- 原试卷保持锁定状态，仍作为考试快照和审计依据。
- 新草稿重新进入普通试卷生命周期：编辑 -> 发布校验 -> 发布 -> 被考试引用后锁定。

## 后续任务

- 增加复制弹窗，让教师可在复制前修改新试卷名称和说明。
- 在复制失败时返回更精确的题目不可用明细，方便教师定位已下线或未审核题目。
- 后续引入 `paper_version` 后，复制可选择“按历史快照复制”或“按最新题库版本复制”。
