# 21. 考试发布预检与快照健康

## 目标

本批次强化教师“发布考试”链路。考试发布前必须先通过服务端预检，确认试卷、时间窗口、参考范围、考生快照和试卷快照都可固化，避免发布后才发现没有考生、试卷不可用或快照不完整。

## 后端变更

- 新增接口：`POST /api/exams/preflight`。
- 预检和正式发布共用同一套 `ExamPublishPlan` 校验：
  - 开始时间必须早于结束时间；
  - 考试时长不能超过开始到结束的时间窗口；
  - 试卷必须存在、未删除且已发布；
  - 试卷必须有题目；
  - 试卷题目必须启用且审核通过；
  - 试卷题目必须已绑定有效题目版本快照；
  - 及格线不能高于试卷总分；
  - 考试目标必须在教师授课范围内；
  - 考试目标最终必须解析出至少 1 名有效学生。
- `createExam` 复用预检发布计划，不再单独走一套较弱规则。
- 教师考试列表和详情新增快照健康字段：
  - `candidateSnapshotCount`
  - `questionSnapshotCount`

## 前端变更

- 新增 `preflightExam(payload)`。
- 发布向导从“参考范围”进入“发布确认”时自动调用服务端预检。
- 预检失败时停留在当前步骤，并展示服务端错误。
- 确认页展示服务端将固化的考生数和题目数。
- 考试列表新增“快照”列，显示已固化的考生快照数和题目快照数。

## 三端协作影响

- 教师端发布前能看到实际锁定的考生和题目数量。
- 学生端进入考试时仍基于 `exam_candidate_snapshot` 和 `exam_question_snapshot`，不再依赖实时班级或题库状态。
- 管理员端后续可基于快照健康字段做全局异常监控，比如“0 考生快照”或“0 题目快照”的考试。

## 后续任务

- 在预检结果中返回每个目标解析出的学生数量，帮助教师发现空课程班。
- 增加正式的发布审批状态：`DRAFT -> PENDING_APPROVAL -> PUBLISHED -> CLOSED`。
- 增加考试创建后快照完整性定时巡检和管理员告警。
