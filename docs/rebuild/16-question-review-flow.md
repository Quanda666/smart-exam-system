# 第十六批落地：题库审核流

## 本批目标

把题库从“教师直接发布即可组卷”升级为“草稿 -> 提交审核 -> 审核通过 -> 可组卷”的治理流程，避免未审核、未完善或被驳回的题目进入试卷。

## 已落地内容

- `question` 新增 `review_status`、`reviewed_by`、`reviewed_at`、`review_comment` 字段。
- 旧库启动迁移会自动补列、补索引，并将历史 `status=1` 的题回填为 `APPROVED`。
- 新增接口：
  - `POST /api/questions/{id}/review/submit`
  - `POST /api/questions/{id}/review/approve`
  - `POST /api/questions/{id}/review/reject`
- 新建和编辑题目统一回到草稿并下架，不能绕过审核直接进入可组卷状态。
- 提交审核和审核通过前会校验题目完整性：客观题正确选项数量合法，填空/主观题有参考答案。
- 自动组卷和手动组卷都只允许引用 `status=1` 且 `review_status=APPROVED` 的题目。
- 题库页面改为显示审核状态和审核意见，操作从“发布/撤回”调整为“提交审核/通过/驳回/下架”。
- 题库统计卡片改为展示题目总数、可组卷、待审核、已驳回。
- 题库导出新增审核状态和审核意见字段，便于质检和批量治理。

## 后续待做

- 增加细粒度权限动作，例如 `QUESTION_REVIEW_APPROVE`，避免所有教师天然拥有审核入口。
- 增加题目版本表，审核通过后的题目被编辑时保留旧版本，已发布考试继续引用快照。
- 增加审核队列页面，支持按学科、题型、来源、创建人筛选待审题。
- 增加批量审核日志和操作审计，支撑正式教学场景的责任追溯。
