# 04 管理员端开发说明

## 必补功能

- 组织架构：学院、专业、年级、行政班、课程班。
- 用户管理：批量导入学生、批量导入教师、教师审核、账号冻结、重置密码。
- 权限管理：角色、页面权限、操作权限、数据范围模板。
- 基础数据：学期、课程、科目、知识点、授课分配、学生选课。
- 考试监管：考试列表、进行中考试、考生进度、异常事件、强制收卷。
- 系统配置：登录策略、密码策略、考试默认规则、AI 开关、邮件配置。
- 数据治理：备份、恢复、导入导出、清理历史草稿、归档旧考试。

## 页面结构

- 工作台：系统概览、今日考试、待审核教师、异常事件、容量状态。
- 组织与用户：组织树、用户列表、导入任务、审核任务。
- 教学基础：课程、课程班、授课、选课、知识点。
- 考试中心：全部考试、监控看板、成绩发布、异常处理。
- 系统管理：角色权限、系统配置、日志审计、备份恢复。

## 数据协同

- 管理员维护课程班后，教师端发布考试只能选择可见课程班。
- 管理员启用教师账号后，教师才可登录并创建题目、试卷、考试。
- 管理员调整授课关系后，教师数据范围和学生考试范围必须实时生效。
- 管理员撤回成绩后，学生端成绩详情应立即不可见，但教师端保留评分记录。

## 验收标准

- 管理员可以从空库完成组织、用户、课程班、授课、选课初始化。
- 导入失败要显示行号、字段、失败原因，不允许静默丢数据。
- 任意管理员高风险操作都能在操作日志中追踪。
