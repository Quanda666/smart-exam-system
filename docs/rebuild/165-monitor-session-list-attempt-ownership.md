# 165. 监考会话列表按答卷归属加载

## 背景

第 164 批修复了监考处置权限加载，但教师监考看板和会话导出仍直接读取 `exam_monitor_session.exam_id/user_id`。如果 session 快照被历史数据或外部写入污染，看板显示、风险排序、学生范围过滤可能和处置权限不一致。

本批把监考会话列表也统一到答卷归属：

- `exam_monitor_session.attempt_id` 仍是会话主锚点。
- 考试、学生、考生快照、学生档案和教学范围过滤均从 `exam_attempt` 推导。

## 本批改动

- `listExamMonitorSessions` 返回的 `examId/userId` 改为 `exam_attempt.exam_id/user_id`。
- 考试表、学生表、考生快照、学生档案关联改为使用答卷归属。
- 会话列表按 `a.exam_id = ?` 过滤考试。
- 非管理员/非考试创建者的学生范围过滤改为 `a.user_id IN (...)`。
- 质量门禁固定监考看板列表必须按答卷归属加载。

## 三端协同影响

- 教师端：监考看板显示的学生、班级、风险和状态与真实答卷一致。
- 管理员端：监考会话导出不会因 session 快照脏数据挂错考试或学生。
- 学生端：无交互变化，但监考处置和通知链路的数据来源更一致。

## 验收点

- 会话列表 `examId` 来自 `exam_attempt.exam_id`。
- 会话列表 `userId` 来自 `exam_attempt.user_id`。
- 考生快照按答卷考试和学生关联。
- 教学范围过滤按答卷学生过滤。
- 处置加载和看板列表使用同一归属规则。
