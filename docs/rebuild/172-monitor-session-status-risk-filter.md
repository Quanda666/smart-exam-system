# 172. 监考会话状态与风险筛选

## 背景

监考事件已经支持按风险分查看和导出，但教师进入考试监控时，第一步通常不是逐条看事件，而是先定位“在线异常”“已交卷”“高风险”的学生会话。

此前监考会话表格只能展示全量会话。考试人数较多时，教师需要手动扫表，定位效率低。本批在教师端看板增加本地状态/风险筛选，不扩大接口面，先把已有 `status/riskScore/riskLevel` 字段用起来。

## 本批改动

- 监考会话看板新增状态筛选：
  - `All status`
  - `Online`
  - `Offline`
  - `Submitted`
- 新增会话风险筛选：
  - `All risk`
  - `Risk > 0`
  - `Risk >= warningThreshold`
  - `Risk >= highThreshold`
- 表格数据源从 `sessions` 改为 `filteredSessions`。
- 指标卡片按筛选后的会话重新计算。
- 筛选条显示当前匹配数量：`filtered/total`。
- 筛选后无匹配时显示空状态提示。
- 新增 `resetSessionFilters`。
- 质量门固定会话筛选控件、过滤数据源和过滤后指标口径。

## 三端协同影响

- 学生端：无变化。
- 教师端：可先按状态/风险定位学生，再进入事件明细筛选单条事件。
- 管理员端：后续可以沿用同一筛选模型做全局监控页或后端分页筛选。

## 验收点

- 会话表格使用 `filteredSessions`。
- 状态筛选可过滤在线、离线、已交卷。
- 风险筛选可过滤有风险、预警、高风险会话。
- 指标卡片跟随筛选变化。
- 重置按钮恢复全量会话。
