# 181. 监考会话未读提醒指标

## 背景

第 180 批已经支持按最近通知状态筛选会话。教师在筛选前仍需要先知道当前列表里是否存在“提醒已发但学生未读”的情况，因此监考面板需要把未读提醒数量提升到指标区。

## 本批改动

- 监考面板指标区新增 `Unread notices`。
- 指标基于当前筛选后的会话计算。
- 仅统计最近处置已发送通知且通知未读的会话。
- `latestNotificationStatusMatches` 复用 `hasLatestNotification`，避免发送状态判断分叉。
- 质量门增加未读提醒指标、计算字段和筛选复用断言。

## 三端协同影响

- 教师端：进入监考看板后能直接看到当前筛选结果内还有多少提醒未被学生读取。
- 学生端：通知已读状态仍由学生查看通知更新，不改变作答流程。
- 管理员端：与第 179、180 批导出的通知状态字段一致，便于核对监考提醒触达情况。

## 验收点

- 最近通知已发送且未读的会话计入 `Unread notices`。
- 最近通知已读的会话不计入。
- 最近处置无通知的会话不计入。
- 改变会话筛选条件后，指标随 `filteredSessions` 同步变化。
