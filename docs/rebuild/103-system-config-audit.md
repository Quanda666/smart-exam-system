# 103. 系统配置变更审计

## 背景

系统配置会影响考试运行的关键行为，例如草稿缓存、监考风险阈值、成绩申诉窗口、审批提醒周期、测试夹具开关等。管理员可以修改这些参数，但如果没有变更审计，后续排查“为什么考试行为变化”“谁调整了监考阈值”“申诉窗口为何改变”时只能依赖数据库快照或人工记忆。

本批为系统配置增加正式变更审计：每次配置值真实发生变化时，记录旧值、新值、分类、类型、操作人和时间，并提供管理员查询与导出。

## 本次改动

- 新增数据库表 `system_config_log`：
  - `config_key`
  - `old_value`
  - `new_value`
  - `value_type`
  - `category`
  - `actor_id`
  - `created_at`
- `DatabaseMigrationRunner` 增加旧库自愈建表逻辑。
- `SystemConfigService.updateConfig` 在配置值变化时写入审计日志。
- 新增系统配置审计查询接口：
  - `GET /api/system/configs/audit`
- 新增系统配置审计导出接口：
  - `GET /api/system/configs/audit/export`
- 查询筛选条件包括：
  - `keyword`
  - `category`
  - `configKey`
  - `actorId`
  - `startFrom/startTo`
- 前端系统配置页新增 `Audit` 抽屉：
  - 展示配置项、分类、旧值、新值、操作人和时间。
  - 支持筛选、分页和导出。
- 本地质量门覆盖迁移表、后端记录/查询/导出、前端 API 和页面入口。

## 三端协同

- 管理员端负责维护配置，并能追踪每次配置变更。
- 教师端和学生端不直接修改配置，但考试、监考、阅卷、成绩申诉等行为会受配置影响。
- 出现运行争议时，管理员可以按配置项或时间窗口导出变更记录，与考试/监考/通知/成绩审计一起复盘。
- 配置审计只记录已发生的配置变更，不替代审批流程；后续可以在高风险配置上叠加变更审批。

## 验收点

- 修改可编辑配置后，`system_config_log` 写入旧值和新值。
- 未发生值变化的保存不会产生重复审计噪音。
- 管理员可以在系统配置页打开 `Audit` 抽屉。
- 可以按配置项、分类、关键字和时间范围筛选审计记录。
- 可以导出当前筛选条件下的审计 CSV。
- 导出最多 5000 条，避免一次性导出拖垮系统。
- 非管理员仍不能访问系统配置和配置审计接口。
