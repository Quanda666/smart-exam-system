# 99. 监考事件明细导出

## 背景

第 98 批已经支持按考试导出监考会话汇总，但会话汇总只能说明学生风险分、事件数量和最近处置。正式考试复盘时，教师或管理员还需要导出单个答卷的原始事件明细，例如切屏、窗口失焦、复制、粘贴、退出全屏、断网和心跳失败等事件发生时间与客户端事件 ID。

## 本次改动

- 新增监考事件明细导出接口：
  - `GET /api/monitor/cheat-events/{attemptId}/export`
- 导出接口复用现有答卷监考事件权限：
  - 管理员可导出全局答卷事件。
  - 考试创建者可导出本考试答卷事件。
  - 非创建者教师只能导出自己授课范围内学生的答卷事件。
- 顺手收紧 `requireAttemptAccess`：
  - 不再只按考试范围放行。
  - 会绑定答卷学生，确保非创建者教师不能查看或导出非授课范围学生的监考事件。
- 导出字段包括：
  - 考试
  - 学生
  - 学号
  - 班级
  - 事件类型
  - 客户端事件 ID
  - 客户端时间
  - 服务端时间
  - 附加信息
  - 事件 ID
  - 答卷 ID
- 前端实时监考事件抽屉新增“导出事件”按钮。

## 协同关系

- 学生端批量上报监考事件。
- 后端用 `clientEventId` 去重，避免网络重试重复累计风险分。
- 教师在实时监考页查看事件明细并可导出原始事件证据。
- 事件明细仍只作为风险记录，不自动判定违规；最终违规处理必须由人工依据制度确认。

## 验收点

- 打开某个学生的监考事件抽屉后可以导出事件明细 CSV。
- 导出结果包含客户端事件 ID、客户端时间和服务端接收时间。
- 非授权教师不能导出不属于自己授课范围学生的监考事件。
- CSV 文件中文在 Excel 中可正常显示。
- 本地质量门禁覆盖后端导出接口、权限收口和前端导出按钮。
