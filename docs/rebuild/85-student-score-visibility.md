# 85. 学生成绩可见性状态

## 背景

教师端和管理员端已经形成成绩发布/撤回审计闭环，但学生端原先只返回已发布成绩。成绩撤回后，学生成绩列表会直接消失对应记录，无法区分“未批阅”“未发布”“已撤回”，也不利于后续申诉和通知解释。

## 本次改动

- 学生成绩列表返回已提交、待批阅、已完成答卷记录。
- 未发布或已撤回成绩不返回分数。
- `GradeInfo` 新增：
  - `scoreVisible`
  - `scoreVisibility`
  - `scoreReleaseStatus`
  - `scorePublishedAt`
  - `scoreRevokedAt`
  - `scoreRevokeReason`
- 成绩详情接口继续只允许已发布成绩访问。
- 学生端成绩页新增“成绩状态”列。
- 已发布成绩显示发布时间，可查看详情和申诉。
- 已撤回成绩显示撤回原因，禁用详情入口。
- 待批阅、待判分、待发布分别给出状态说明。

## 状态语义

- `RELEASED`：成绩已发布，学生可查看分数和答题详情。
- `REVOKED`：成绩曾发布后被撤回，学生不可查看分数和详情。
- `PENDING_REVIEW`：答卷仍在教师批阅中。
- `PENDING_FINALIZE`：答卷已提交但还未形成最终成绩。
- `PENDING_RELEASE`：成绩已形成但教师尚未发布。

## 三端协同意义

- 教师端：发布/撤回动作会影响学生端可见状态。
- 管理员端：全局审计记录可解释学生端看到的撤回原因。
- 学生端：不再出现“记录消失”或“空分数不明原因”的体验断裂。

## 验收点

- 未发布成绩不暴露分数。
- 已撤回成绩不暴露分数，但展示撤回原因。
- 待批阅答卷显示“待批阅”，不能查看详情。
- 已发布成绩可查看分数、答题详情和申诉入口。
- 本地质量门禁通过。
