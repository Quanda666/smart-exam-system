# 184. 监考通知审计 ID 复制

## 背景

教师端监考面板已经展示 `Notification #id`，管理员端通知审计也已经支持按 `Notification ID` 精确检索。缺口在于教师需要人工抄录 ID，跨端协同时容易出错。

## 本批改动

- 教师端会话列表的最新处置通知增加 `Copy audit ID` 按钮。
- 教师端处置时间线的每条通知处置记录增加 `Copy audit ID` 按钮。
- 复制内容只包含通知 ID，便于管理员直接粘贴到 `Notification Audit` 的 `Notification ID` 输入框。
- 复制逻辑优先使用 `navigator.clipboard.writeText`，并保留 textarea 后备方案。
- 质量门禁增加复制按钮、图标、剪贴板函数和时间线布局断言。

## 三端协同影响

- 教师端：发现通知未读、通知异常或学生反馈未收到时，可以复制审计 ID 给管理员。
- 管理员端：使用教师提供的 ID 在通知审计里精确定位发送、已读、创建时间等记录。
- 学生端：不改变通知读取与考试作答流程；通知仍然只是监考沟通与风险处置记录。

## 验收点

- 会话列表存在最新通知 ID 时显示 `Copy audit ID`。
- 处置时间线存在通知 ID 时显示 `Copy audit ID`。
- 点击后剪贴板内容为纯通知 ID。
- 没有通知 ID 的记录不显示复制按钮。
- 既有通知筛选、导出、未读统计不受影响。
