## 项目定位

智慧在线考试与学习反馈系统面向高校课程考核、课堂测验和阶段训练，覆盖基础数据、题库、试卷、考试、阅卷、成绩、错题、分析、通知、日志和 AI 辅助教学。

## 当前功能

### 管理员端

- 概况：系统用户、题库、试卷、考试、日志和基础数据总览。
- 基础数据：班级、课程、课程班、授课分配、学生归属、科目、知识点、公告。
- 业务管理：题库、试卷、考试任务、成绩分析。
- 系统管理：用户、角色、系统日志、防作弊事件。

### 教师端

- 概况：待处理考试、待阅卷、题库和学生反馈总览。
- 题库管理：题目维护、发布撤回、AI 出题台。
- AI 出题台：直接生成题目、上传题目文档识别、上传课程资料按题型生成题目。
- 试卷考试：手动组卷、规则组卷、发布考试、阅卷和 AI 评分建议。
- 学情分析：成绩统计、知识点掌握度、学生画像。

### 学生端

- 概况：考试、成绩、错题和通知总览。
- 考试中心：查看考试、在线答题、保存草稿、提交试卷。
- 学习反馈：成绩记录、错题本、知识点掌握度、错题 AI 讲解。
- 基础信息：公告和个人资料。

## 技术架构

| 层次 | 说明 |
|---|---|
| 前端 | Vue 3 + Vite + TypeScript + Element Plus + ECharts |
| 后端 | Java 17 + Spring Boot 3 + JDBC + AOP + Bean Validation |
| 数据库 | MySQL 8 |
| AI | OpenAI 兼容 Chat Completions 接口，支持模拟模式和本地规则兜底 |
| 部署 | Docker 多阶段构建，前端构建产物打入 Spring Boot Jar |

## 数据库

数据库以 `backend/src/main/resources/db/schema.sql` 和 `backend/src/main/resources/db/data.sql` 为唯一权威来源。后端启动时自动建表和写入初始化数据。

主要表包括：

- 用户与角色：`sys_user`、`sys_role`、`sys_user_role`
- 基础数据：`edu_class`、`edu_subject`、`edu_course`、`class_course`、`teacher_class_course`、`student_class_membership`
- 题库试卷：`question`、`question_option`、`paper`、`paper_question`
- 考试阅卷：`exam`、`exam_target`、`exam_attempt`、`exam_answer_draft`、`answer_record`、`review_record`
- 反馈分析：`wrong_question_book`、`cheat_event`、`operation_log`、`notification`
- AI 与邮件：`ai_prompt_template`、`ai_usage_log`、`email_verification`、`user_token`

## AI 设计

AI 能力遵循“辅助生成，人工确认”的原则：

- 教师直接生成结构化题目草稿。
- 上传题目文档后自动识别题干、选项、答案和解析。
- 上传课程资料后按题型数量生成关联知识点题目。
- 教师阅卷时获取主观题评分建议。
- 学生错题本中获取个性化讲解。

AI 结果不会自动发布题目，也不会直接决定主观题成绩。

## 部署和运行

开发运行：

```bash
cd backend && mvn spring-boot:run
cd frontend && npm run dev
```

Windows 一体化运行：

```cmd
scripts\run-all.cmd
```

本地验收见 `docs/deploy-local.md`，云部署见 `docs/deploy-railway.md` 和 `docs/deploy-render.md`。
